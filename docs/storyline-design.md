## 四、数据文件约定

- **剧集列表**：`backend/data/topics.json`（中文）、`backend/data/topics-en.json`（英文），字段含 `topic_title`、`topic_description`、`phase`、`emotional_theme`、`core_points`。
- **角色**：`backend/data/agents.json`、`backend/data/agents-en.json`，保持 `id` 不变以保证与既有对话数据兼容。
- **单集对话**：`backend/data/conversations/{date}.json` 与 `{date}-en.json` 的顶层需包含 `topic_title`、`topic_description`、`phase`，建议与当日 `topics` 一致。

## 五、戏剧张力类型（供生成参考）

- **时间压力**：限时决策（如氧气、日落前完成）。
- **资源稀缺**：只能做 A 或 B，无法兼顾。
- **信息不对称**：某人知情未说或系统异常原因不明。
- **信任危机**：专业判断被质疑。
- **价值冲突**：技术最优 vs 人的需求（如安全 vs 探索）。

## 六、每日对话流程（供生成参考）

1. **情境导入（3–5 分钟）**：由计算机/物理科学家描述当日情境；科幻作家给出情感框架。
2. **技术辩论（15–20 分钟）**：问题定义 → 方案提出 → 交叉质询 → 综合与决策。
3. **叙事收尾（3–5 分钟）**：科幻作家「今日叙事」；系统状态更新；情感收尾。

更完整的角色提示词、叙事风格、错误纠正与可读性检查见原始设计方案。

### 使用脚本生成指定日

```bash
# 依赖
pip install openai

# 使用本地 Ollama（需先启动 ollama serve）
export OPENAI_API_BASE=http://localhost:11434/v1
export OPENAI_API_KEY=ollama
export OPENAI_MODEL=llama3.2   # 或其他已拉取模型

python scripts/generate_story_day.py 2026-03-07   # 生成第 3 天「呼吸之间」
python scripts/generate_story_day.py 2026-03-08 --overwrite   # 重新生成并覆盖
```

使用 OpenAI 或其它兼容 API 时，设置 `OPENAI_API_KEY` 和（可选）`OPENAI_API_BASE`、`OPENAI_MODEL` 即可。提示词在 `backend/data/prompts/system_storyline.txt`，可按需修改。
